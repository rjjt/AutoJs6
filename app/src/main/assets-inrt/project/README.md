# 模拟拉人 — QQ 群邀请自动化

> 装到安卓机上，自动从你的群里点"邀请 → 从群聊邀请 → 我加入的群 → 目标群"批量勾选成员邀请。

## 安装

1. 从 GitHub Releases 下载最新的 `模拟拉人-debug-buildN.apk`
2. 安卓机允许"安装未知来源应用"
3. 装上，打开应用

## 首次配置

打开应用后填写：

| 字段 | 说明 |
|---|---|
| 源群名称 | 你自己的群（被邀请人会进这里） |
| 目标群 | 每行一个，按顺序处理 |
| 每批勾选数 | 一次邀请操作勾选多少人。腾讯单次邀请上限 30 |
| 批次间隔 | 每批结束后等多久再下一批，分钟 |
| 每日上限 | 每天最多邀请总人数 |
| 操作延迟 | 每次点击/滑动之间的随机延迟，秒 |
| 凌晨保护 | 0-7 点禁用（异常时段更易触发风控） |

## 权限

- **无障碍服务**：脚本通过这个识别 QQ 的 UI 控件
- **悬浮窗权限**：运行时显示状态/控制按钮在 QQ 上层
- **建议关闭电池优化**：避免后台运行被系统杀掉

应用内"检查权限"按钮会提示当前状态。

## 使用流程

1. 在应用里填好配置，点"保存"
2. 手动打开 QQ，登录账号，找到源群（先不要进去）
3. 切回应用，点"开始"
4. 切到 QQ，悬浮窗会跟着你 → 脚本接管操作

悬浮窗按钮：

- ▶ / ‖ — 暂停/继续
- ■ — 停止
- ⚙ — 占位（暂未实现回到设置页）

## 已知风险

- **腾讯风控**：QQ 对自动化邀请检测严格，频繁/大批量会限制邀请、冻结、封号
- **协议风险**：批量邀请可能违反《腾讯 QQ 软件许可及服务协议》，使用者自负
- **建议**：先小号、小批量（每天 < 30 人）测试一周以上，再考虑放量

## 开发

源码在 `app/src/main/assets-inrt/project/`：

```
project/
├── project.json     # 应用元信息
├── main.js          # 入口
├── core/
│   ├── storage.js   # 配置和今日计数持久化
│   ├── anti.js      # 随机延迟、凌晨保护
│   ├── nav.js       # QQ 导航（步骤 1-7）
│   ├── select.js    # 成员勾选（步骤 8-9）
│   └── engine.js    # 状态机（开始/暂停/停止）
└── ui/
    ├── settings.js  # 设置界面
    └── floating.js  # 悬浮窗
```

修改任意 `.js` 文件后 `git push`，GitHub Actions 自动出新 APK 在 Releases。
